Design principles

  • Separation of concerns — audio/transcription logic stays in VoiceToText; agent logic will live in separate modules.
  • Incremental build — start with a working transcription path, then layer agents on top of stable text output.
  • Session-bound mic — user picks an input device once at startup; the same device_index is used for the whole run.
  • macOS target — microphone I/O via PortAudio (Homebrew); verify with brew list portaudio.
  • Isolated Python env — use .venv/ for dependencies and runs; avoid global pip installs.
  • Voice command — after a pause, say send to press Enter immediately (priority queue); no . after send until dictation resumes.
  • Session prefix — after mic pick + click in typing window, type @voice-input-confirmation.mds ; send types it again after Enter.
  • Pause punctuation — silence > 2 s → . + Shift+Enter; Enter alone is only via send.
  • Cursor output — transcripts go to the focused app via _type_at_cursor() / pyautogui; Terminal shows status and errors only.
  • Living documentation — root-level docs are updated whenever the codebase changes (see sync-root-docs.mds).
